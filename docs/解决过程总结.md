# Save to Feishu 故障排查与优化总结

## 问题概述
- 保存失败：字段 ID 不存在或字段映射失效
- 公众号图片抓取失败或画质较差
- 图文顺序错乱
- 保存速度慢
- 保存中按钮反馈不明显

## 排查思路与解决方案

### 1. 字段映射失效导致保存失败
**现象**
- 前端提示：字段 ID 不存在，要求重新获取字段

**原因定位**
- 表格字段发生变更后，本地缓存的 fieldId 已失效

**解决方案**
- 在保存时用“字段名”回退匹配当前表格字段
- 匹配成功时自动更新本地映射，避免下次再次失败
- 匹配失败时跳过该字段，保证记录仍可保存

### 2. 公众号图片获取不到或画质差
**现象**
- 图片经常抓不到或是占位图
- 图片清晰度不佳

**原因定位**
- 公众号使用懒加载，真实图片在 data-src / data-original 等属性
- 图片域名（qpic/qlogo）未在 host_permissions 中声明，下载失败

**解决方案**
- 优先读取真实图片地址（data-src 等）
- 规范化图片 URL（处理 // 前缀、&amp; ）
- 增加 qpic/qlogo 域名权限

### 3. 图文顺序错乱
**现象**
- 图片被追加到文末，顺序与原文不一致

**原因定位**
- 图片上传时重新创建图片块，导致插入位置不正确

**解决方案**
- 先按原文顺序写入“占位图片块”
- 写入文档后收集创建块 ID
- 上传图片时更新对应占位块

### 4. 保存速度慢
**现象**
- 长文或多图保存时耗时明显

**原因定位**
- 图片上传是串行执行，且每张图片固定等待
- 写入块有批次延迟

**解决方案**
- 图片上传改为并发（限制并发数为 2）
- 轻量间隔启动，避免频控但不强制长等待

### 5. 保存中反馈不明显
**现象**
- 用户点击后感觉没有反应

**解决方案**
- 按钮保存中状态添加：脉冲光晕 + 闪光扫过 + 图标旋转
- 保存开始/结束时切换动画状态

## 当前状态
- 保存失败与字段映射问题已自动兼容
- 公众号图片抓取与权限已优化
- 图文顺序正确，图片嵌入稳定
- 保存速度显著提升
- 保存中按钮视觉反馈明显

## 建议的发布前检查
- 重新加载扩展后，使用公众号文章做完整保存测试
- 检查多图文章的顺序与清晰度
- 检查字段映射自动修复是否生效
